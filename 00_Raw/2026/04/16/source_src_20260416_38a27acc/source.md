# 로컬 서버 데스크톱 앱 패키징 기법

- created_at: 2026-04-16T00:35:45.241Z
- knowledge_type: development-code

## Raw Text
새로운 지식 노드를 추가하고 싶어. 주제는 "로컬(Local) 서버를 데스크톱 네이티브 앱처럼 패키징하고 바탕화면에 예쁘게 전시하는 3단계 기법"이야.

보통 Node.js나 Vite로 만든 로컬 웹 서비스는 터미널을 열고 'npm run dev'를 친 다음, 브라우저를 켜서 localhost:5173으로 접속해야 하는 귀찮은 과정이 있어. 나는 이걸 사용자가 클릭 한 번으로 실행할 수 있는 "진짜 컴퓨터 프로그램"처럼 속이고(?) 싶거든. 이 기법은 다음 과정을 거쳐.

1. PWA(Progressive Web App) 위장술
- public 폴더에 manifest.json과 sw.js(서비스 워커)를 추가해. 브라우저 주소창이나 탭을 없애고(standalone 모드) 앱처럼 띄우기 위한 껍데기 작업이야.
- 192x192, 512x512 크기의 png 아이콘도 필요해.

2. Batch 매크로 스크립트(.bat) 생성
- 사용자가 직접 터미널을 치지 않게 해주는 스크립트야.
- 배치 파일의 실행 순서는 이래:
  가. 프로젝트 폴더로 cd 이동
  나. start /min cmd /c "npm run dev" 명령어로 터미널 창을 백그라운드 구석(최소화)에 숨긴 채 서버를 켬.
  다. timeout /t 3 으로 서버가 켜질 시간을 줌.
  라. 가장 중요한 핵심! `start chrome --app=http://localhost:5173/` 명령어를 써서, 브라우저를 주소창 없는 깔끔한 윈도우 창 모드(App 모드)로 열어줌.

3. 화룡점정: 예쁜 아이콘 바로가기(.lnk) 만들기
- .bat 파일은 윈도우의 기본 톱니바퀴 아이콘밖에 못 가져서 안 예뻐.
- 그래서 PowerShell 스크립트(`WScript.Shell`)를 써서 바탕화면에 새로운 윈도우 바로가기(.lnk)를 만들어. 이 바로가기의 타겟(Target)은 방금 만든 .bat 파일로 연결하고, 아이콘(IconLocation)은 우리가 만든 예쁜 .ico 파일로 입혀버리는 거야.
- 이러면 사용자는 톱니바퀴 대신 예쁜 아이콘만 바탕화면에 보게 되고, 그걸 더블클릭하면 진짜 완성된 데스크톱 프로그램을 쓰는 것 같은 경험을 하게 돼!
